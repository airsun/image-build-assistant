# Gemini 多模态使用模式

dan-ultra-workflow 前端编码阶段使用的 Gemini Pro 多模态模式。每种模式适用于不同场景，编排 skill 根据 task 元数据选择对应模式。

---

## 1. Design-to-Code

**场景：** task 有 `[design:path/to/mockup.png]` 标签

**操作步骤：**
1. 提示用户在 Cursor 中打开设计稿图片
2. 在 Composer 中引用设计稿图片 + spec 相关段落
3. Prompt 框架：
   ```
   请根据这个设计稿实现对应的前端组件。

   设计稿：[附图]
   功能规格：{{spec_excerpt}}
   技术要求：
   - 使用 {{framework}} 框架
   - 遵循项目现有组件结构
   - 响应式适配
   ```
4. 产出代码后进入 Screenshot-Verify 验证

---

## 2. Screenshot-Verify

**场景：** 前端编码完成后验证实现效果

**操作步骤：**
1. 使用 Browser MCP 启动本地开发服务器（如已运行则跳过）
2. 导航到目标页面
3. 截取页面截屏
4. 将截屏与 spec/设计稿对比：
   ```
   对比截屏和功能规格，检查：
   1. 布局是否与设计一致
   2. 交互元素是否齐全
   3. 文案是否正确
   4. 响应式表现（可选：截取不同视口宽度）
   ```
5. 有偏差 → 修正代码 → 重新截屏验证

---

## 3. Visual-Diff-Loop

**场景：** 复杂前端组件需要多轮迭代（常与 Ralph-loop 配合）

**操作步骤：**
1. 实现初版代码
2. 截屏当前状态
3. 与目标设计对比，列出差异清单
4. 逐项修正 → 截屏 → 对比
5. 循环直到视觉差异为零或可接受

**Ralph-loop 配合：**
将差异清单写入 scratchpad prompt，Ralph 自动驱动迭代。

---

## 4. Spec-Diagram-Read

**场景：** spec 或 design 中包含流程图、架构图等图表

**操作步骤：**
1. 将图表文件路径传入 Gemini
2. Prompt 框架：
   ```
   请解读这张图表并提取：
   1. 涉及的实体和关系
   2. 数据流向
   3. 关键状态转换
   4. 对应的代码结构建议
   ```
3. 将 Gemini 解读结果作为编码上下文

---

## 5. Component-Consistency

**场景：** 确保新组件与项目现有组件风格一致

**操作步骤：**
1. 截屏项目中已有的类似组件
2. 截屏新开发的组件
3. 对比：
   ```
   对比这两个组件截屏：
   - 左图：项目现有组件（参考风格）
   - 右图：新开发的组件

   检查以下一致性：
   1. 颜色方案
   2. 字体大小和间距
   3. 圆角、阴影等视觉细节
   4. 交互态（hover/active/disabled）的风格
   ```
4. 列出不一致项 → 修正

---

## 6. Responsive-Check

**场景：** 验证页面在不同设备宽度下的表现

**操作步骤：**
1. 使用 Browser MCP 设置不同视口宽度：
   - 移动端：375px
   - 平板：768px
   - 桌面：1280px
2. 逐个宽度截屏
3. 对比检查：
   ```
   检查这 3 张不同视口宽度的截屏：
   1. 375px（移动端）
   2. 768px（平板）
   3. 1280px（桌面）

   验证：
   - 布局是否合理适配
   - 文字是否溢出或过小
   - 交互元素是否可点击（尺寸足够）
   - 导航是否正确折叠/展开
   ```
4. 有问题 → 修正 → 重新截屏验证

---

## 模式选择速查

| Task 特征 | 推荐模式 |
|-----------|----------|
| 有 `[design:...]` | Design-to-Code → Screenshot-Verify |
| 无设计稿的前端 task | 直接编码 → Screenshot-Verify |
| `[complex]` 前端 | Visual-Diff-Loop（建议 Ralph-loop） |
| spec 含图表 | Spec-Diagram-Read |
| 新增组件 | Component-Consistency |
| 任何前端完成后 | Responsive-Check（可选） |
